# Тест-кейсы API qa-internship.avito.com

## Обозначения

- **P** — позитивный сценарий
- **N** — негативный сценарий
- **C** — корнер-кейс
- **NF** — нефункциональная проверка

## Техники тест-дизайна

- **Эквивалентное разбиение**: валидные/невалидные значения для `sellerID`, `price`, `name`
- **Граничные значения**: `price: 0`, `sellerID: 111111` и `999999`, пустая строка для `name`
- **Классы эквивалентности**: числовые / строковые / пустые значения для параметров
- **Попарное тестирование**: комбинации наличия/отсутствия полей в теле запроса
- **Идемпотентность**: GET-запросы возвращают одинаковый результат; POST создаёт новый ресурс каждый раз
