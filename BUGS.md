# Баг-репорты API qa-internship.avito.com

---

## BUG-1: API отклоняет создание объявления с price=0

**Серьёзность:** Medium

**Краткое описание:**
При создании объявления с `price: 0` API возвращает 400 Bad Request с сообщением «поле price обязательно», хотя 0 является валидным целым числом. API трактует значение `0` как отсутствующее (falsy).

**Шаги воспроизведения:**
1. Отправить POST `/api/1/item` с телом:
   ```json
   { "sellerID": 222222, "name": "Бесплатный товар", "price": 0, "statistics": { "contacts": 1, "likes": 1, "viewCount": 1 } }
   ```

**Фактический результат:**
HTTP 400, тело: `{"result":{"message":"поле price обязательно","messages":{}},"status":"400"}`

**Ожидаемый результат:**
HTTP 200, объявление успешно создано с ценой 0.

**Окружение:**
- Host: https://qa-internship.avito.com
- Endpoint: POST /api/1/item

## BUG-2: API аналогично отклоняет statistics с нулевыми значениями

**Серьёзность:** Medium

**Краткое описание:**
При создании объявления со `statistics: { contacts: 0, likes: 0, viewCount: 0 }` API возвращает 400 с сообщением «поле likes обязательно». Значение `0` воспринимается как отсутствие поля.

**Шаги воспроизведения:**
1. Отправить POST `/api/1/item` с телом:
   ```json
   { "sellerID": 222222, "name": "Тест", "price": 100, "statistics": { "contacts": 0, "likes": 0, "viewCount": 0 } }
   ```

**Фактический результат:**
HTTP 400, тело: `{"result":{"message":"поле likes обязательно","messages":{}},"status":"400"}`

**Ожидаемый результат:**
HTTP 200, объявление создано. Значение `0` — допустимое целое число.

**Окружение:**
- Host: https://qa-internship.avito.com
- Endpoint: POST /api/1/item

## BUG-3: API принимает отрицательную цену при создании объявления

**Серьёзность:** High

**Краткое описание:**
При создании объявления с `price: -100` API возвращает 200 и успешно сохраняет объявление. Отрицательная цена не имеет бизнес-смысла и должна отклоняться валидацией.

**Шаги воспроизведения:**
1. Отправить POST `/api/1/item` с телом:
   ```json
   { "sellerID": 222222, "name": "Тест", "price": -100, "statistics": { "contacts": 1, "likes": 1, "viewCount": 1 } }
   ```

**Фактический результат:**
HTTP 200, объявление создано с отрицательной ценой.

**Ожидаемый результат:**
HTTP 400, ошибка валидации — цена не может быть отрицательной.

**Окружение:**
- Host: https://qa-internship.avito.com
- Endpoint: POST /api/1/item

## BUG-4: API принимает отрицательный sellerID при запросе объявлений продавца

**Серьёзность:** Medium

**Краткое описание:**
GET `/api/1/-1/item` возвращает 200 OK вместо 400 Bad Request. Отрицательный sellerID не имеет смысла и должен отклоняться.

**Шаги воспроизведения:**
1. Отправить GET `/api/1/-1/item`

**Фактический результат:**
HTTP 200, возвращается массив (возможно пустой или с данными).

**Ожидаемый результат:**
HTTP 400 Bad Request — невалидный формат sellerID.

**Окружение:**
- Host: https://qa-internship.avito.com
- Endpoint: GET /api/1/:sellerID/item

## BUG-5: GET /api/2/statistic/:id возвращает 404 вместо 400 для невалидного формата ID

**Серьёзность:** Low

**Краткое описание:**
При запросе статистики с невалидным форматом ID (не UUID) эндпоинт v2 возвращает 404 Not Found, тогда как корректный ответ — 400 Bad Request. Аналогичный эндпоинт v1 (`/api/1/statistic/:id`) корректно возвращает 400.

**Шаги воспроизведения:**
1. Отправить GET `/api/2/statistic/invalid`

**Фактический результат:**
HTTP 404 Not Found.

**Ожидаемый результат:**
HTTP 400 Bad Request — невалидный формат идентификатора.

**Окружение:**
- Host: https://qa-internship.avito.com
- Endpoint: GET /api/2/statistic/:id

## BUG-6: Формат ответа POST /api/1/item не соответствует спецификации Postman

**Серьёзность:** Medium

**Краткое описание:**
Согласно Postman-коллекции, при успешном создании объявления ответ должен содержать объект с полями `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`. Фактически API возвращает только строку `{ "status": "Сохранили объявление - {uuid}" }`.

**Шаги воспроизведения:**
1. Отправить POST `/api/1/item` с валидными данными.

**Фактический результат:**
```json
{ "status": "Сохранили объявление - f1d233c2-1dff-4571-8b8a-25f0ee65ad1b" }
```

**Ожидаемый результат (по Postman-спецификации):**
```json
{
  "id": "f1d233c2-1dff-4571-8b8a-25f0ee65ad1b",
  "sellerId": 222222,
  "name": "Тест",
  "price": 100,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 1 },
  "createdAt": "2026-04-05T..."
}
```

**Окружение:**
- Host: https://qa-internship.avito.com
- Endpoint: POST /api/1/item
